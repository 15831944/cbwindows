How Hidden Pieces Work

First some teminology is needed.

When you create pieces in the game designer program you can chose one or two images for pieces. There is a FRONT tile image and an optional BACK image.

When you play a game with two sided pieces a piece has a normally visible TOP side and the face down BOTTOM side.

IMPORTANT RULE: If a piece is considered hidden (for whatever reason) during game play the FRONT image is displayed. This is true regardless of what image of piece is on TOP.

You also need to understand the notion of game players. Players can OWN things used in a game. I'll cleverly call them the OWNERs. The following things can be OWNed (or not OWNed) when playing in CyberBoard's multiplayer mode:

  • Piece TRAYs
  • Playing BOARDs
  • Pieces

IMPORTANT RULE: The OWNERship of a BOARD or a TRAY cannot be changed in a game. The OWNERship of pieces can change.

IMPORTANT RULE: If a TRAY or BOARD is OWNed, then all the pieces residing in or on them are automatically set to be OWNed by that TRAY or BOARD's OWNER.

IMPORTANT RULE: Pieces in non-OWNed TRAYs cannot be OWNed by anyone. Those pieces will have their OWNERship automatically cleared.

IMPORTANT RULE: Pieces on non-OWNed BOARDs can either be OWNed or not OWNed. Non-OWNed BOARDs will probably be where most of the action takes place.

Therefore, non-OWNERs of pieces will only see the FRONT tile image of them during play. OWNERs will see whatever image is on TOP.

Visibility of Tray Content

Trays have several visibility rules available:

  1. Both sides of pieces are fully visible
  2. Only the top side of pieces are visible
  3. Each piece image is hidden. (shows "HIDDEN")
  4. All piece items hidden. (shows the text "ALL HIDDEN")

I'll call these the TRAY VISIBILITY RULES.

If a TRAY is OWNed by the current player, then rule (1) is used *unless* the "Visibility rule also applies to the tray's owner" option is checked.

If a TRAY is OWNed but not by the current player, then the TRAY VISIBILITY RULES are used.

If a TRAY isn't OWNed by anyone, then the TRAY VISIBILITY RULES are used.

Security of Pieces, Boards, and Trays

There is a single option which selects how secure TRAYs, BOARDs, and PIECES are from your opponents. The Option is set in either the BOARD properties or the TRAY properties dialogs. The checkbox option is:

"Allow non-owners to access pieces on this board/tray."

I'll call this the ACCESS OPTION.

If the ACCESS OPTION is cleared for OWNed TRAYs and BOARDs, those BOARDs and TRAYs are completely inaccessable to Non-OWNERs. They operate normally for the OWNers. If the ACCESS OPTION is set for this case, then only the visibility rules are enforced.

If the ACCESS OPTION is cleared for non-OWNed BOARDs, then pieces that are OWNed by players other than the current player cannot selected or otherwise affected. If the ACCESS OPTION is set for this case, then only the visibility rules are enforced.

Some Miscellaneous Information

If a move affects a piece in a manner that should be hidden from view, the move is played back SILENTLY. The opponent is not be aware anything happened. The following operations performed on OWNed pieces will not be seen by your opponent:

  • Flipping a piece over
  • Setting a piece's text
  • Locking or unlocking a Piece

At times it can be confusing to know what your opponent can actually see since you, as the owner, see quite a bit more. To see the boards and trays from the perspective of a game spectator use the View/Simulate Specator Player menu command.

Creating a Board Resident Non-Owned Decks of Cards

In CyberBoard cards are simply pieces.

If you need a deck of cards that reside on a BOARD (rather than in a TRAY) and aren't owned by any player, a special option has been added to assist in this situation. Basically what is needed is that when a non-OWNed card is selected on a BOARD or viewed in a TRAY, the BOTTOM information should NOT be visible. The checkbox option is specified when a piece is created and reads as follows:

"Only show top image during games."

IMPORTANT RULE: This rule is only enforced if a piece is not OWNed. You can force the rule to apply to everyone including OWNERs by checking the following companion option:

"Enforce above option for owned pieces too."

How to Create a Multiplayer Game

First you need to create a multiplayer Scenario. This is done using the menu command Project/Create Players.... If this is set to a non-zero value, the Scenario is considered to be multiplayer and supports hidden units.

You can change the default player names using the menu command Project/Edit Players....

Set ownership and security on various TRAYs and BOARDs as required.

Create a game as you normally would and select the multiplayer scenario file. First you will be given the opportunity to change the player names to real names. Press Ok. Next you select a base filename for the game. This name will be the root part of all the generated player files. When you press Ok the player files are created. The following files will be created:

  • RootFilename-<Player_Name>.gam. One of these will be created for each player in the game. These should be EMailed to the appropriate players.
  • RootFilename-Spectator.gam. This file has no OWNERship access rights. It can be sent to spectators who will observe the game.
  • RootFilename-Referee.gam. This file has total rights. Generally the Referee for a game should only have this file. This game file allows access and viewability to everything. It could also be used for "power" spectators who would like to look over the shoulder of all other players.
